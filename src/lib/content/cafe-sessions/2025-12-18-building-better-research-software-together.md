## Session Summary

This session focused on practical software engineering habits that help research teams write code that is easier to understand, maintain, and reuse.

## Topics Covered

- Organizing projects with a predictable folder and module structure.
- Using Git workflows to keep history clear and collaboration smoother.
- Writing minimal documentation so others can run and extend the work.
- Improving reproducibility with pinned environments and scripted steps.
